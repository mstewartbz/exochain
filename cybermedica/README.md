# CyberMedica QMS Contracts

CyberMedica is an adjacent clinical research QMS surface. It is not Exochain core and does not modify Exochain source. This package establishes baseline service contracts for governed QMS workflows, evidence receipts, production trust activation, fail-closed adapter responses, and inactive trust-state UI models.

Production Exochain/root-backed trust claims remain inactive unless the activation gates in `docs/context/CYBERMEDICA_PRODUCTION_TRUST_ACTIVATION_GATES.md` are satisfied by verified runtime evidence.

## Current Build

| Area | Status |
|---|---|
| Surface classification | Adjacent surface |
| Runtime stack | Node ESM service contracts |
| Exochain source | Read-only evidence only |
| Production trust state | Inactive by default |
| Secrets | None required for baseline contracts |
| Test command | `npm test` |
| Coverage command | `npm run test:coverage` |
| Quality gate | `npm run quality` |

## Trust Boundary

CyberMedica stores mutable operational state. Exochain receipts, DAG/provenance, root bundles, gateway/node decisions, and Decision Forum outcomes remain external trust evidence until a verified adapter path is selected and tested. The baseline contracts do not mint, simulate, cache, or override consent, authority, quorum, provenance, root authority, or Decision Forum outcomes, and production activation evidence is denied if it is locally simulated, cached, overridden, or carries protected content or secret material.
