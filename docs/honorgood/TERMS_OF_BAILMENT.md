# Terms Of Bailment

HonorGood supports bailment-style terms as deterministic records. This does not assert that every digital contribution is legally a bailment in every jurisdiction. It records a custody and use wrapper with permitted purpose, attribution, settlement, audit, revocation, and dispute handling.

## Terms Record

`BailmentTerms` records:

- bailor reference
- bailee reference policy
- contribution node
- permitted use
- prohibited use
- custody scope
- attribution requirement
- settlement requirement
- beneficiary reference
- revocation, dispute, and audit policy references
- jurisdiction reference
- human-approval triggers
- whether agent execution is allowed

Ownership is not transferred unless a separate accepted or ratified instrument says so. The adopter receives only the permitted use and custody scope recorded in the accepted terms.

## Human Approval

Human approval is required for new legal templates, disputes, revocation, off-policy use, high-risk custody, and materiality exceptions. Routine settlement can be automated only when accepted terms, authority, value events, and rulesets are all valid.
