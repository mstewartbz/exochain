## What's solid

- No `unsafe` blocks anywhere in `crates/` — `unsafe_code = "deny"` enforced at workspace level.
- Identity/authorization: centralized default-deny gate, correct key-rotation semantics, strong negative-test coverage (quorum−1/quorum/quorum+1, challenge-blocked quorum, revoked-key rejection, multibase prefix validation).
- Deterministic-types contract (`exo-core`) forbids `HashMap` and floats; uses `BTreeMap` + BLAKE3 + CBOR.
- `workspace.lints` denies `float_arithmetic`, `float_cmp`, `unsafe_code`.
- `deny.toml` carefully written with rationale for every ignored advisory.
- No production secrets committed — only clearly-labeled dev defaults.
- `deploy/Dockerfile.node` is correctly hardened (non-root, exec `ENTRYPOINT`, `HEALTHCHECK`).
- GAP-REGISTRY well-structured (7 closed with commit SHAs, 3 open with blockers).
- CI has 22 blocking gates covering build, test, 90% coverage, clippy `-D warnings`, fmt, audit `--deny warnings`, deny, doc, hygiene, SBOM, machete, integration tests (gateway / DB / consensus / state-sync), cross-platform build, 0dentity module coverage (≥80%), WASM build + bridge verification + export-count sync.
