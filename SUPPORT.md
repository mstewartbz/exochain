# Support

## Getting Help

### Documentation

Start with the [docs/INDEX.md](docs/INDEX.md) for the full documentation index, including:

- [Getting Started Guide](docs/guides/GETTING-STARTED.md) — Build, test, and understand the constitutional constraints
- [Architecture](docs/architecture/ARCHITECTURE.md) — System overview, dependency graph, design rationale
- [Crate Reference](docs/reference/CRATE-REFERENCE.md) — API reference for all 16 crates

### Issues

For **bug reports** and **feature requests**, use the structured GitHub Issue templates:

- [Bug Report](https://github.com/exochain/exochain/issues/new?template=bug_report.yml)
- [Feature Request](https://github.com/exochain/exochain/issues/new?template=feature_request.yml)

Issues labeled `exoforge:triage` are automatically routed through the ExoForge self-improvement cycle for council review.

### Discussions

For questions, ideas, and general discussion, use [GitHub Discussions](https://github.com/exochain/exochain/discussions) (when enabled).

## Security Issues

**Do not open public issues for security vulnerabilities.** See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full contribution workflow, including:

- Code review requirements (mapped to council panels via CODEOWNERS)
- CI quality gates that must pass before merge
- ExoForge self-improvement workflow for AI-assisted development

## Commercial Support

EXOCHAIN is pre-release open-source software licensed under Apache-2.0. Commercial support inquiries: **support@exochain.org**

## Response Expectations

This is a pre-release project maintained by a small team. Please set expectations accordingly:

- **Security issues**: Acknowledged within 48 hours (see SECURITY.md)
- **Bug reports**: Triaged within 1 week
- **Feature requests**: Reviewed as capacity allows; ExoForge triage may propose implementation plans
- **General questions**: Best-effort response
