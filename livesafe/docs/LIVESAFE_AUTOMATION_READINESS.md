## Safe Automation Scope Now

A LiveSafe automation can safely perform these tasks now:

- Normalize new context dumps into `context/canon`.
- Maintain repository and artifact maps.
- Compare imported claims against local repo evidence.
- Add deterministic tests for boundary evaluation and context validation.
- Build local contract modules that do not process raw sensitive data.
- Use synthetic fixtures for card, emergency profile, contacts, PACE roles,
  responder role, vault references, and QR pointer behavior.
- Build entitlement and onboarding contracts with synthetic users, plans,
  cohorts, gifts, trials, and marketplace templates.
- Build storage entitlement contracts with synthetic storage levels, encrypted
  blobs, content-addressed references, safe anchor fields, quotas, and billing
  states.
- Build AI help and feedback contracts with synthetic sessions, redacted
  metadata, deterministic help topics, mandated reporter thresholds, and
  disabled-by-default agent dispatch.
- Build legacy-adjacent product contracts only as inactive projections with
  synthetic fixtures and explicit `exo-legacy` dependency state.
- Keep EXOCHAIN trust state inactive until verified runtime evidence exists.
- Trust ExoForge for internal development planning, implementation workflow,
  review routing, and validation support during genesis.
- Build trust-signal output contracts that make unverified status obvious at a
  glance without making public trust claims.
- Build consent and revocation receipt contracts that stay adapter-gated,
  metadata-only, and inactive until EXOCHAIN proof paths are verified.
- Build homologated trust-display contracts for jurisdiction, language, script,
  mobile, tablet, desktop, print, API, and holonic contexts.
- Update docs only when new source-backed facts or completed behavior exists.

## Unsafe Automation Scope Now

A LiveSafe automation should not yet do these tasks:

- Claim production EXOCHAIN enforcement.
- Import raw medical, identity, guardian, trustee, location, QR, contact, or
  vault data from old artifacts.
- Treat obsolete phone numbers, URLs, or QR targets as current endpoints.
- Copy historical legal, medical-release, transfer, or rights-assertion text
  into a current product policy without a current source and version.
- Build a live first-responder disclosure route before the emergency-profile,
  authorization, access-scope, and revocation contracts exist.
- Store raw medical records, genetic profile data, eligibility documents, or
  payment secrets in repo fixtures, logs, or generated artifacts.
- Store raw medical, genetic, identity, contact, emergency, or vault material in
  IPFS, Filecoin, object storage, logs, fixtures, provider metadata, or
  EXOCHAIN anchors.
- Treat trial matching, frontline eligibility, family-plan benefits, or
  marketplace capability behavior as active until the entitlement and consent
  contracts exist.
- Dispatch autonomous coding agents directly from AI help sessions without
  feedback triage, feature gates, rate limits, and redacted payloads.
- Represent legacy charter, genetic unveiling, mausoleum export, erasure,
  posthumous-representation, or self-retirement capabilities as active before
  `exo-legacy` exists and a verified adapter passes.
- Modify EXOCHAIN core.
- Signal public or customer-facing trust from ExoForge output, root ceremony
  intent, or development progress before internal proof gates pass.
- Render trust-bearing output without the required AVC badge, symbol, color,
  glow, text, and machine-readable status.
- Treat localization as translation only when trust-state meaning, jurisdiction,
  script, cultural-symbol review, accessibility, and layout stability are also
  required.
